# معرض الصور - Photo Gallery Script

سكربت PHP كامل لمعرض ألبومات صور، بتصميم أسود/فضي/ذهبي، مع ألبومات قابلة للحماية بكلمة مرور، ولوحة تحكم كاملة الصلاحيات.

---

## 1. المتطلبات

قبل الرفع، تأكد أن استضافتك توفر:

- **PHP 8.0 أو أحدث** (الكود يستخدم صياغة `match` المتوفرة من PHP 8 فقط)
- **MySQL 5.7+ أو MariaDB 10.3+**
- امتداد **PDO_MySQL** (مُفعّل افتراضياً في أغلب الاستضافات)
- امتداد **GD** لمعالجة الصور وإنشاء الصور المصغرة (مُفعّل افتراضياً في أغلب الاستضافات)
- صلاحية الكتابة (Write permission) على مجلد `uploads/`

أغلب استضافات cPanel المشتركة (Hostinger, Namecheap, GoDaddy, إلخ) تدعم هذه المتطلبات مباشرة. تأكد فقط من اختيار **PHP 8.x** من قائمة "Select PHP Version" في لوحة تحكم الاستضافة إن كانت تعرض خيارات متعددة.

---

## 2. خطوات التثبيت

### أ) رفع الملفات
ارفع كل محتويات المجلد إلى مجلد موقعك على الاستضافة (عادة `public_html` أو `public_html/اسم-مجلد-فرعي` إن أردت تثبيته في مسار فرعي).

### ب) إنشاء قاعدة البيانات
1. من لوحة تحكم الاستضافة (cPanel عادة)، أنشئ قاعدة بيانات MySQL جديدة ومستخدم بصلاحيات كاملة عليها.
2. افتح **phpMyAdmin**، اختر قاعدة البيانات الجديدة، ثم من تبويب **Import** ارفع ملف `database.sql` الموجود في جذر المشروع.
3. تحقق أن الجداول أُنشئت بنجاح (10 جداول: admins, albums, photos... إلخ).

### ج) ربط إعدادات الاتصال
افتح ملف `config/database.php` وعدّل القيم التالية ببيانات استضافتك الفعلية:

```php
define('DB_HOST', 'localhost');       // غالباً localhost
define('DB_NAME', 'اسم_قاعدة_البيانات');
define('DB_USER', 'اسم_المستخدم');
define('DB_PASS', 'كلمة_مرور_قاعدة_البيانات');

define('SITE_URL', 'https://yourdomain.com');   // رابط موقعك الفعلي بدون / في النهاية
```

كذلك يُفضّل تغيير `SECRET_KEY` إلى قيمة عشوائية خاصة بك.

### د) صلاحيات مجلد الرفع
تأكد أن مجلد `uploads/` (وما بداخله) قابل للكتابة من قبل السيرفر. غالباً `755` تكفي، وإن ظهرت مشاكل في الرفع جرّب `775`.

### هـ) تسجيل الدخول لأول مرة
افتح: `https://yourdomain.com/admin/login.php`

**بيانات الدخول الافتراضية:**
```
اسم المستخدم: admin
كلمة المرور:  Admin@12345
```

⚠️ **غيّر كلمة المرور فوراً** بعد أول تسجيل دخول من: لوحة التحكم ← الملف الشخصي ← تغيير كلمة المرور.

### و) تنظيف ما بعد التثبيت (مهم للأمان)
- احذف ملف `database.sql` من السيرفر بعد الاستيراد الناجح (أو تأكد أن ملف `.htaccess` المرفق يحجبه — راجع قسم الأمان أدناه).
- لا تترك أي نسخة احتياطية من قاعدة البيانات داخل مجلد يمكن الوصول إليه عبر المتصفح.

---

## 3. إن لم تعمل بيانات الدخول الافتراضية

نادراً ما يحدث تعارض في تنسيق تشفير كلمة المرور بين بيئات مختلفة. إن رفضت الصفحة الدخول ببيانات الدخول الافتراضية:

**إن كانت استضافتك تدعم SSH:**
```bash
php cli-reset-password.php
```
اتبع التعليمات لإدخال اسم مستخدم وكلمة مرور جديدة. هذه الأداة تعمل فقط عبر الطرفية ولا يمكن الوصول إليها من المتصفح إطلاقاً (آمنة تماماً لتركها على السيرفر).

**إن لم تكن تدعم SSH:**
1. أنشئ ملف مؤقت باسم `temp-hash.php` بهذا المحتوى:
   ```php
   <?php echo password_hash('كلمة_المرور_الجديدة', PASSWORD_DEFAULT);
   ```
2. ارفعه لجذر الموقع وافتحه من المتصفح مرة واحدة، وانسخ الناتج (نص طويل يبدأ بـ `$2y$`).
3. من phpMyAdmin، افتح جدول `admins`، وعدّل قيمة حقل `password` للمستخدم `admin` بالصق الناتج.
4. **احذف `temp-hash.php` فوراً** بعد الانتهاء.

## 4. ملاحظات أمنية مهمة

- **الحماية بكلمة مرور للألبومات** تمنع تصفح الألبوم أو معرفة محتواه من واجهة الموقع. مع ذلك، روابط الصور المباشرة تُنشأ بأسماء عشوائية غير قابلة للتخمين، لكنها ليست معزولة بالكامل خلف تحقق الجلسة على مستوى الملفات نفسها (أي أن من يحصل على رابط صورة مباشر واحد من متصفح فتح الألبوم بالفعل يمكنه الوصول لتلك الصورة تحديداً دون كلمة المرور). هذا مستوى حماية مناسب للخصوصية العامة (منع التصفح العشوائي)، وليس حماية بمستوى بيانات حساسة للغاية. إن كنت بحاجة لحماية أكثر صرامة (تحقق من الجلسة عند كل طلب صورة)، أخبرني ويمكنني تطوير نسخة أكثر تشدداً.
- ملفات `.htaccess` المرفقة تمنع الوصول المباشر لمجلدي `config/` و`includes/`، وتمنع تنفيذ أي كود داخل `uploads/`. هذه الحماية تعمل على خوادم **Apache** فقط. إن كانت استضافتك تستخدم **Nginx**، أخبرني لأزودك بإعدادات مكافئة.
- غيّر `SECRET_KEY` في `config/database.php` قبل الاستخدام الفعلي.
- يُنصح بشدة بتفعيل **HTTPS** (شهادة SSL) خصوصاً لصفحة تسجيل الدخول وكلمات مرور الألبومات.

## 5. هيكل المشروع

```
├── admin/                  # لوحة التحكم
│   ├── ajax/                  # نقاط AJAX (ترتيب الصور بالسحب)
│   ├── includes/               # هيدر/فوتر لوحة التحكم
│   ├── index.php               # لوحة القيادة
│   ├── albums.php / album-edit.php / album-photos.php
│   ├── categories.php
│   ├── admins.php / admin-edit.php
│   ├── settings.php / appearance.php / profile.php
│   └── activity-log.php / photos.php
├── assets/
│   ├── css/ (style.css للموقع، admin.css للوحة التحكم)
│   └── js/  (main.js للموقع، admin.js للوحة التحكم)
├── config/database.php     # إعدادات الاتصال (عدّله أولاً)
├── includes/               # النماذج والدوال المساعدة (Album, Photo, ImageHandler...)
├── uploads/                # مجلد الصور المرفوعة (يُنشأ تلقائياً حسب الألبوم)
├── database.sql            # هيكل قاعدة البيانات + بيانات أولية
├── cli-reset-password.php  # أداة طوارئ لإعادة تعيين كلمة مرور المدير (CLI فقط)
├── index.php / albums.php / album.php / about.php / contact.php  # صفحات الموقع العامة
└── download.php            # تحميل الصور الأصلية (يحترم الحماية والإعدادات)
```

## 6. الصلاحيات المتوفرة في لوحة التحكم

عند إضافة مشرف جديد (غير "مدير عام")، يمكنك منحه أي مجموعة من هذه الصلاحيات بدقة من صفحة "المشرفون":

| الصلاحية | الوصف |
|---|---|
| إدارة الألبومات | إنشاء/تعديل الألبومات ورفع الصور |
| حذف الألبومات | صلاحية منفصلة لحذف الألبومات نهائياً |
| إدارة التصنيفات | إضافة/تعديل/حذف التصنيفات |
| إدارة المشرفين | إضافة مشرفين جدد وتعديل صلاحياتهم |
| إعدادات الموقع | تعديل الإعدادات العامة والمظهر |
| سجل النشاطات | الاطلاع على سجل كل العمليات |
| إدارة التعليقات | الموافقة على التعليقات أو حذفها |

"المدير العام" (Super Admin) يملك كل الصلاحيات تلقائياً ولا يمكن لأي مشرف آخر تعديل حسابه ما لم يكن هو نفسه مديراً عاماً، حمايةً من تصعيد الصلاحيات.

## 7. المميزات المضمّنة

- ألبومات متعددة، كل واحد قابل للحماية بكلمة مرور مستقلة
- تصنيفات، ترتيب مخصص، تمييز ألبومات معينة
- رفع صور متعدد بالسحب والإفلات + إنشاء صور مصغرة تلقائياً
- إعادة ترتيب الصور بالسحب داخل الألبوم + تحديد صورة الغلاف
- صندوق عرض صور (Lightbox) مع تنقل بالأسهم ولوحة المفاتيح + تحميل الصورة الأصلية
- تعليقات الزوار على الألبومات مع نظام موافقة من لوحة التحكم (قابل للتفعيل/الإيقاف)
- علامة مائية نصية اختيارية تُضاف تلقائياً على الصور الجديدة
- نظام صلاحيات متعدد المستويات (مدير عام / مشرف / محرر) مع تخصيص دقيق
- سجل نشاطات كامل لكل تعديل يحدث في لوحة التحكم
- 3 ثيمات لونية (ذهبي/فضي/برونزي) قابلة للتبديل من لوحة التحكم دون لمس الكود
- حماية CSRF على كل النماذج، وحماية من محاولات تخمين كلمات المرور المتكررة
- تصميم متجاوب بالكامل مع الجوال
